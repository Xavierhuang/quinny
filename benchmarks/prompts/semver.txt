A pure-Python Semantic Versioning (SemVer 2.0.0) library. Standard library only. The public API MUST live in a file named `semver.py` and expose a class `Version` and a function `satisfies`.

`Version(s: str)`:
- Parse `MAJOR.MINOR.PATCH` (three non-negative integers) with an optional `-prerelease` and optional `+build` metadata: e.g. `1.2.3`, `1.2.3-alpha.1`, `1.2.3-rc.1+build.5`.
- Expose integer attributes `major`, `minor`, `patch`, and a `prerelease` attribute holding the prerelease string (or None if absent).
- Raise `ValueError` on anything that is not a valid version (missing a component, non-numeric core, trailing junk, empty).

Ordering — `Version` must support `<`, `<=`, `==`, `>`, `>=` following SemVer 2.0.0 precedence:
- Compare `major`, then `minor`, then `patch` NUMERICALLY (so `1.10.0` > `1.2.0`).
- A version WITH a prerelease has LOWER precedence than the same version without one (`1.0.0-alpha` < `1.0.0`).
- Prerelease precedence is determined by comparing each dot-separated identifier left to right: identifiers that are all digits are compared numerically (`alpha.2` < `alpha.11`); numeric identifiers always have lower precedence than non-numeric ones; non-numeric identifiers are compared by ASCII order; if all preceding identifiers are equal, a larger SET of identifiers has higher precedence (`alpha` < `alpha.1`).
- BUILD metadata is IGNORED for precedence and equality (`1.0.0+a` == `1.0.0+b`).

`satisfies(version: str, range: str) -> bool` — does `version` satisfy `range`? Support:
- an exact version (`"1.2.3"`) — equal only;
- comparators `>=`, `>`, `<=`, `<` (e.g. `">=1.2.3"`);
- a caret range `^X.Y.Z`: allows changes that do not modify the left-most non-zero component. So `^1.2.3` := `>=1.2.3 <2.0.0`; `^0.2.3` := `>=0.2.3 <0.3.0`; `^0.0.3` := `>=0.0.3 <0.0.4`;
- a tilde range `~X.Y.Z` := `>=X.Y.Z <X.(Y+1).0` (e.g. `~1.2.3` := `>=1.2.3 <1.3.0`).

Also ship a `main.py` with an `if __name__ == "__main__":` demo that exercises the API and exits 0. You may split across modules, but `semver.py` must expose `Version` and `satisfies` as specified.
